# Linux/UNIX Command Reference

리눅스/유닉스 환경에서 자주 사용되는 필수 명령어들을 정리한 문서입니다. 초보자부터 중급자까지 모두에게 유용한 명령어들을 기능별로 분류했습니다.

## 🌟 1. 리눅스 사용자를 위한 꿀팁 (Pro-tips)

본격적인 명령어 학습에 앞서, 생산성을 극대화할 수 있는 몇 가지 팁입니다.

| 기능 | 설명 | 예시 |
| :--- | :--- | :--- |
| **Tab 자동 완성** | 명령어, 파일명, 경로 입력 중 `Tab` 키를 누르면 자동으로 완성해줍니다. 리눅스 사용의 **필수 스킬**입니다. | `cd /var/l` + `Tab` → `cd /var/log/` |
| **`history`** | 이전에 사용했던 명령어 기록을 보여줍니다. `!번호`로 해당 명령어를 다시 실행할 수 있습니다. | `history`, `!102` |
| **파이프 `\|`** | 한 명령어의 결과를 다른 명령어의 입력으로 전달합니다. 명령어들을 조합하여 강력한 기능을 만듭니다. | `ps aux \| grep "nginx"` |
| **리다이렉션 `>` & `>>`** | 명령어의 결과를 화면 대신 파일로 저장합니다. `>`는 덮어쓰기, `>>`는 이어쓰기입니다. | `ls -l > list.txt` |
| **`man [명령어]`** | 명령어의 공식 매뉴얼(설명서)을 보여줍니다. 가장 신뢰할 수 있는 정보 소스입니다. | `man ls` |
| **`sudo`** | 관리자(root) 권한으로 명령어를 실행합니다. 시스템 설정 변경이나 패키지 설치 시 필요합니다. | `sudo apt update` |

## 🚀 2. 기본 명령어 (초보자 필수)

### 2.1 파일 및 디렉토리 관리

| 명령어 | 설명 | 주요 옵션 및 예시 |
| :--- | :--- | :--- |
| **`ls`** | 현재 디렉토리의 파일/폴더 목록을 보여줍니다. | **-l**: 상세 정보 표시<br>**-a**: 숨김 파일 포함<br>**-h**: 파일 크기를 읽기 쉬운 단위로<br>**-t**: 수정한 시간 순으로 정렬<br>_`ls -alth` (가장 많이 쓰는 조합)_ |
| **`cd`** | 디렉토리를 이동합니다. | `cd [경로]`: 지정 경로로 이동<br>`cd .`: 현재 디렉토리<br>`cd ..`: 상위 디렉토리<br>`cd ~` 또는 `cd`: 홈 디렉토리<br>`cd -`: 바로 이전 디렉토리 |
| **`pwd`** | 현재 작업 중인 디렉토리의 전체 경로를 보여줍니다. | (옵션 없이 단독 사용) |
| **`mkdir`** | 새 디렉토리를 생성합니다. | **-p**: 하위 디렉토리까지 한 번에 생성<br>_`mkdir -p project/src`_ |
| **`rm`** | 파일이나 디렉토리를 삭제합니다. **(경고: 휴지통 없이 영구 삭제!)** | **-r**: 디렉토리와 그 안의 내용까지 삭제<br>**-f**: 묻지 않고 강제 삭제<br>_`rm -rf old_project`_ |
| **`cp`** | 파일이나 디렉토리를 복사합니다. | **-r**: 디렉토리를 복사할 때 사용<br>_`cp main.c backup/main_v1.c`_ |
| **`mv`** | 파일/디렉토리를 이동시키거나 이름을 변경합니다. | _`mv old.txt new.txt` (이름 변경)_<br>_`mv report.pdf ./documents/` (이동)_ |
| **`touch`** | 빈 파일을 생성하거나, 파일의 최종 수정 시간을 업데이트합니다. | _`touch new_file.txt`_ |

### 2.2 파일 내용 확인

| 명령어 | 설명 | 중요 옵션 및 예시 |
| :--- | :--- | :--- |
| **`cat`** | 파일 내용을 터미널에 출력합니다. (짧은 파일용) | _`cat config.txt`_ |
| **`less`** | 긴 파일 내용을 페이지 단위로 편하게 봅니다. (스크롤, 검색 가능, `q`로 종료) | _`less /var/log/syslog`_ |
| **`head`** | 파일의 앞부분 몇 줄을 보여줍니다. (기본 10줄) | **-n [숫자]**: 지정한 줄 수만큼 표시<br>_`head -n 20 log.txt`_ |
| **`tail`** | 파일의 뒷부분 몇 줄을 보여줍니다. (기본 10줄) | **-n [숫자]**: 지정한 줄 수만큼 표시<br>**-f**: 파일에 내용이 추가될 때마다 실시간으로 보여줌 (로그 모니터링 필수!)<br>_`tail -f server.log`_ |

### 2.3 검색

| 명령어 | 설명 | 주요 옵션 및 예시 |
| :--- | :--- | :--- |
| **`find`** | 특정 조건으로 파일을 검색합니다. | `find [경로] -name "[파일명]"`: 이름으로 검색<br>`find . -type f -mtime -7`: 7일 내 수정된 파일 검색<br>_`find . -name "*.py"`_ |
| **`grep`** | 파일 내용 중에서 특정 문자열(패턴)이 포함된 줄을 찾습니다. | **-r**: 디렉토리 하위의 모든 파일에서 검색<br>**-i**: 대소문자 구분 안 함<br>**-v**: 특정 패턴을 제외한 결과 출력<br>**-n**: 줄 번호와 함께 출력<br>_`grep -rin "error" ./src`_ |

### 2.4 시스템 및 권한

| 명령어 | 설명 | 주요 옵션 및 예시 |
| :--- | :--- | :--- |
| **`ps`** | 현재 실행 중인 프로세스 목록을 보여줍니다. | **aux**: 시스템의 모든 프로세스를 상세히 표시<br>**ef**: 프로세스 관계를 트리 구조로 표시<br>_`ps aux \| grep "python"`_ |
| **`kill`** | 프로세스를 종료시킵니다. | `kill [PID]`: 정상 종료 시도 (SIGTERM)<br>`kill -9 [PID]`: 강제 종료 (SIGKILL)<br>_`kill -9 12345`_ |
| **`chmod`** | 파일/디렉토리의 권한(읽기, 쓰기, 실행)을 변경합니다. | `chmod 755 script.sh` (rwx, r-x, r-x)<br>`chmod u+x script.sh` (소유자에게 실행 권한 추가) |
| **`df`** | 디스크의 남은 용량을 보여줍니다. | **-h**: 읽기 쉬운 단위(G, M)로 표시<br>_`df -h`_ |
| **`du`** | 특정 파일/디렉토리의 디스크 사용량을 보여줍니다. | **-h**: 읽기 쉬운 단위로 표시<br>**-s**: 총 사용량만 요약<br>_`du -sh /var/log`_ |

## 💎 3. 중급/고급 명령어 (알아두면 전문가)

### 3.1 네트워킹

| 명령어 | 설명 | 주요 옵션 및 예시 |
| :--- | :--- | :--- |
| **`ssh`** | 원격 서버에 안전하게 접속합니다. (Secure Shell) | `ssh [사용자명]@[호스트주소]`<br>`ssh -p [포트번호] user@host`<br>_`ssh ubuntu@192.168.1.100`_ |
| **`scp`** | `ssh`를 이용해 원격 서버와 파일을 안전하게 주고받습니다. | `scp [파일명] [원격지 ID]@[원격지 주소]:[경로]` (업로드)<br>`scp user@host:/remote/file .` (다운로드)<br>**-r**: 디렉토리 전체를 복사 |
| **`wget`** | 웹에서 파일을 다운로드합니다. | **-c**: 이어받기<br>**-O [파일명]**: 다른 이름으로 저장<br>_`wget -c https://.../archive.zip`_ |
| **`curl`** | 다양한 프로토콜로 데이터를 전송합니다. (API 테스트 등에 매우 유용) | `curl -L [URL]`: 리다이렉션을 따라가서 요청<br>`curl -X POST -d \'{"key":"val"}\' [URL]`: POST 요청 보내기<br>_`curl https://api.github.com`_ |
| **`netstat`** | 네트워크 연결 상태, 라우팅 테이블, 인터페이스 통계 등을 보여줍니다. | **-anp**: 모든 포트와 연결 상태를 PID와 함께 표시<br>**-lntp**: LISTEN 상태인 TCP 포트를 표시<br>_`sudo netstat -lntp`_ |

### 3.2 텍스트 처리 및 고급 파일 조작

| 명령어 | 설명 | 주요 옵션 및 예시 |
| :--- | :--- | :--- |
| **`sed`** | 스트림 편집기. 파일 내용을 검색, 치환, 추가, 삭제하는 강력한 도구입니다. | `sed \'s/old/new/g\' file.txt`: 파일 내 `old`를 `new`로 모두 치환하여 출력 |
| **`awk`** | 패턴을 스캔하고, 매칭된 패턴에 대해 특정 동작을 수행하는 프로그래밍 언어. (주로 열(column) 기반 데이터 처리에 사용) | `awk \'{print $1, $3}\' data.txt`: `data.txt`에서 첫 번째와 세 번째 열을 출력 |
| **`ln`** | 파일에 대한 링크(바로가기)를 생성합니다. | **-s**: 심볼릭 링크(Symbolic Link) 생성<br>_`ln -s /path/to/original.txt link.txt`_ |

### 3.3 압축 및 아카이빙

| 명령어 | 설명 | 주요 옵션 및 예시 |
| :--- | :--- | :--- |
| **`tar`** | 여러 파일이나 디렉토리를 하나의 파일(`.tar`)로 묶거나 풉니다. | **-c**: 묶기 (Create)<br>**-x**: 풀기 (Extract)<br>**-v**: 처리 과정 표시<br>**-f**: 파일 이름 지정<br>**-z**: gzip 압축/해제 동시 수행 (`.tar.gz`)<br>_`tar -czvf archive.tar.gz ./my_folder` (압축)<br>`tar -xzvf archive.tar.gz` (해제)_ |
| **`zip`** | `.zip` 파일로 압축합니다. | `zip -r archive.zip ./my_folder` |
| **`unzip`** | `.zip` 파일의 압축을 해제합니다. | `unzip archive.zip` |

### 3.4 고급 시스템 및 프로세스 관리

| 명령어 | 설명 | 주요 옵션 및 예시 |
| :--- | :--- | :--- |
| **`chown`** | 파일/디렉토리의 소유자 및 그룹을 변경합니다. | `chown [사용자]:[그룹] [파일]`<br>**-R**: 디렉토리와 하위 항목 모두 변경<br>_`sudo chown -R www-data:www-data /var/www`_ |
| **`htop`** | `top`의 업그레이드 버전. 컬러풀하고 직관적인 인터랙티브 프로세스 뷰어입니다. (별도 설치 필요: `sudo apt install htop`) | `htop` |
| **`crontab`** | 특정 시간에 특정 명령어를 주기적으로 실행하도록 예약합니다. (스케줄링) | **-e**: 스케줄 편집<br>**-l**: 스케줄 목록 보기<br>_`crontab -e` 후 `0 2 * * * /path/to/backup.sh` (매일 새벽 2시에 백업 스크립트 실행)_ |
